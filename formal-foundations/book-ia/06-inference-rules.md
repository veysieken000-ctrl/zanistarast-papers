# Chapter 06 — Inference Rules

Version 1.0

---

# Purpose

This chapter defines the admissible inference rules used throughout the Zanistarast mathematical framework.

No theorem may use an inference rule that has not been explicitly introduced.

---

# Rule IR-1

Definition Expansion

A defined object may be replaced by its formal definition.

---

# Rule IR-2

Function Application

A function may be applied only when its domain conditions are satisfied.

---

# Rule IR-3

Composition

Two functions may be composed only when the codomain of the first matches the domain of the second.

---

# Rule IR-4

Deterministic Preservation

A deterministic transformation preserves previously established deterministic properties.

---

# Rule IR-5

Dependency Preservation

No inference may remove a required dependency from the deterministic chain.

---

# Rule IR-6

Identity Preservation

If every transformation preserves Yek,

their composition preserves Yek.

---

# Rule IR-7

Verification Rule

Rasterast verification may only be concluded after every previous layer has been successfully applied.

---

# Scientific Review Note

These inference rules define how mathematical reasoning proceeds inside the Zanistarast framework.

No proof may use an unstated inference rule.
