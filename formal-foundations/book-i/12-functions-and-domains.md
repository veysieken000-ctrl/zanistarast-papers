# Chapter 12 — Functions and Domains

Version: 1.0

---

# Purpose

This chapter specifies the mathematical nature of the transformations used in the Zanistarast framework.

---

# Partial Functions

The transformations introduced in Book I are modeled as partial functions.

A transformation is defined only for inputs satisfying the required conditions.

---

# Functions

Hebûn : E ⇀ EH

Zanabûn : EH ⇀ K

Mabûn : K ⇀ M

Rabûn : M ⇀ X

Rasterast : X ⇀ VX

where

⇀

denotes a partial function.

---

# Domain Principle

Each transformation is defined only on its valid domain.

Inputs outside the domain are rejected.

---

# Codomain Principle

Every successful transformation produces an output belonging to its codomain.

---

# Composition Principle

The deterministic chain is the composition

Rasterast ∘ Rabûn ∘ Mabûn ∘ Zanabûn ∘ Hebûn

Composition is defined only when every intermediate transformation is defined.

---

# Failure Principle

If one transformation is undefined,

every subsequent transformation is undefined.

---

# Scientific Review Note

Version 1.0 models every transformation as a deterministic partial function.

Future versions may refine these mappings using dependent types or richer formal structures.
