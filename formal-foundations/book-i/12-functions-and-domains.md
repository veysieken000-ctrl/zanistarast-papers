# Functions and Domains

Version: 1.0

Status: Mathematical Functions

# Purpose

Functions establish deterministic transformations between the primitive
mathematical structures of Zanistarast.

Every function preserves ontological consistency.

No function may violate the deterministic dependency chain.

---

# Mathematical Principle

A function maps one well-defined ontological object to another.

Every function has

• a domain,

• a codomain,

• a deterministic rule.

Functions never generate arbitrary objects.

They only reveal mathematically valid relationships.

---

# Domain

A domain is the collection of objects for which a function is defined.

Every domain must belong to the Universe of Discourse.

Undefined objects cannot belong to a domain.

---

# Codomain

The codomain specifies every mathematically possible result of a function.

Every codomain must also belong to the Universe of Discourse.

---

# Identity Function

The identity function preserves ontological identity.

Id(Y)=Y

Identity never creates a new Yek.

It preserves the existing one.

the function always produces identical outputs.

randomness is never a property of the mathematical function itself.

Uncertainty may arise only from incomplete knowledge,

not from the mathematical structure.
