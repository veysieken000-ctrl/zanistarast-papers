# Purpose

This document establishes the metric framework of Zanabûn Mathematics.

Zanabûn Metrics provide deterministic quantitative measures for certified knowledge, semantic consistency, explainability, inference quality, dependency integrity, and compatibility with the Certified Core.

# Definition

Definition 1

A Zanabûn Metric is a deterministic function

d : ℤ × ℤ → ℝ≥0

that evaluates certified knowledge properties.

---

# Knowledge Metric

Notation

dK(z)

Definition

Measures the completeness and determinism of a certified knowledge element.

Evaluation includes

• semantic completeness,

• dependency integrity,

• certification,

• explainability.

---

# Semantic Consistency Metric

Notation

dS(z₁,z₂)

Definition

Measures semantic compatibility between certified knowledge elements.

Smaller values indicate stronger semantic consistency.

Semantic similarity never replaces certified verification.

---

# Explainability Metric

Notation

dE(z)

Definition

Measures the completeness of the reasoning chain supporting a certified knowledge element.

Evaluation includes

• source traceability,

• inference transparency,

• dependency visibility,

• reasoning completeness.

---

# Inference Quality Metric

Notation

dI(z)

Definition

Measures the quality of deterministic inference.

Evaluation includes

• logical validity,

• semantic correctness,

• reproducibility,

• certification success.

---

# Dependency Metric

Notation

dD(z)

Definition

Measures consistency of the knowledge dependency graph.

Maximum consistency requires

• acyclic dependencies,

• deterministic traceability,

• complete dependency reconstruction.

---

# Certification Metric

Notation

dC(z)

Definition

Evaluates compatibility with the Certified Core.

Possible outputs

• Certified

• Under Verification

• Not Certified

Future versions may introduce quantitative certification scores.

---

# Metric Properties

Every certified metric shall satisfy

• determinism,

• reproducibility,

• semantic consistency,

• dependency preservation,

• Rasterast compatibility.

---

# Forbidden Metrics

Metrics shall never

• ignore semantic contradiction,

• bypass certification,

• invalidate dependency,

• generate non-reproducible evaluations,

• replace deterministic verification.
