Version: 1.0

Status: Architecture

# Purpose

This document defines the Project Management subsystem of the Zanistarast Command Line Interface (CLI).

The Project Management subsystem provides deterministic, certified, and reproducible project lifecycle management while preserving dependency integrity, certification continuity, workspace consistency, and compatibility with the Certified Core and Rasterast Mathematics.

Every project operation shall be deterministic, reproducible, explainable, certifiable, and fully auditable.

# Workspace Management

Every workspace contains

• Workspace Identifier

• Project Identifier

• Configuration Reference

• Dependency Graph

• Certification Status

• Audit Reference

# Project Templates

Project templates define

• directory structure,

• configuration schema,

• dependency layout,

• initialization scripts,

• certification metadata.

Only certified templates may be used.

# Project Validation

Every project is validated for

• structural correctness,

• dependency integrity,

• configuration consistency,

• version compatibility,

• Rasterast compatibility.

Projects failing validation are rejected.

Certified project records are immutable.
