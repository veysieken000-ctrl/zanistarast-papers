# Naming Convention Standard

Version: 1.0

Status: Core Review

---

# Purpose

This document establishes the official naming conventions used throughout the Zanistarast framework.

The objective is to ensure consistency across documentation, software, mathematical publications, and future extensions.

A unified naming convention improves readability, traceability, maintainability, and deterministic interpretation.

---

# Scope

This standard defines

• repository naming,

• directory naming,

• document naming,

• source code naming,

• version identifiers,

• commit messages,

• module identifiers.

---

# Repository Naming

Official repositories shall use lowercase characters.

Words shall be separated by hyphens.

Examples

zanistarast-core

zanistarast-runtime

zanistarast-mathematics

zanistarast-specifications

---

# Directory Naming

Directories shall use

lowercase

hyphen-separated words

Examples

core-review

reference-software

scientific-platform

domain-frameworks

future-research

---

# File Naming

Documentation files shall use

two-digit ordering

followed by

descriptive lowercase names.

Examples

01-primitive-definitions.md

02-ehad.md

10-primitive-chain-audit.md

21-terminology-standard.md

No spaces shall appear in filenames.

---

# Source Code Naming

Programming identifiers shall follow the conventions of the implementation language while preserving Zanistarast terminology.

Examples

RasterastValidator

NativeRuntime

CertifiedKnowledge

HebunOperator

---

# Versioning

Official version format

Major.Minor.Patch

Examples

1.0.0

1.1.0

2.0.0

Breaking changes require a new Major version.

---

# Commit Message Format

Commit messages shall follow

type(scope): summary

Examples

docs(core-review): update theorem audit

feat(runtime): implement deterministic scheduler

fix(verification): correct dependency resolution

refactor(core): simplify proof validator

---

# Book Naming

Official publications shall use

Book I

Book II

Book III

...

Book VIII

Additional editions shall append the version number.

Example

Book III v1.2

---

# Phase Naming

Project phases shall use

PHASE A

PHASE B

PHASE C

Subphases shall use

C1

C2

C3

C4

---

# Reserved Terms

The following names are reserved

Ehad

Vahid

Yek

Hebûn

Zanabûn

Mabûn

Rabûn

Rasterast

Certified Core

Native AI Architecture

These names shall not be replaced by unofficial alternatives.
